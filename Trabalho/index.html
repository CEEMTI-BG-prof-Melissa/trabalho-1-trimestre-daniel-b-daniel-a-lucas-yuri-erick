REQUISITOS FUNCIONAIS

  • O sistema deve permitir que usuários realizem cadastro informando dados como nome, e-mail e senha.
  • O sistema deve permitir que usuários autenticados façam login e logout com segurança.
  • O sistema deve disponibilizar uma ferramenta de busca para localizar livros por título, autor ou palavra-chave.
  • O sistema deve permitir a visualização de detalhes de um livro, como sinopse, autor, editora e ano de publicação.
  • O sistema deve permitir que usuários façam o empréstimo digital de livros disponíveis.
  • O sistema deve controlar o prazo de empréstimo e bloquear o acesso ao livro após o vencimento.
  • O sistema deve permitir que usuários façam reservas de livros que estejam indisponíveis no momento.
  • O sistema deve registrar o histórico de leitura e empréstimos de cada usuário.

REQUISITOS NAO FUNCIONAIS

  • Suportar acesso simultâneo de até 5.000 usuários sem falhas.
  • Garantir tempo de carregamento das páginas inferior a 3 segundos.
  • Utilizar criptografia para proteção de dados sensíveis dos usuários.
  • Assegurar conformidade com diretrizes de acessibilidade digital.
  • Possuir mecanismos de recuperação automática em caso de falhas.
  • Permitir manutenção sem interrupção total do serviço.
  • Manter registros de atividades (logs) por no mínimo 6 meses.
  • Oferecer escalabilidade para suportar aumento de demanda.

REQUISITOS INVERSOS

  • Não permitir a exclusão definitiva: O sistema não deve permitir que usuários (incluindo bibliotecários) excluam permanentemente registros bibliográficos ou documentos digitais; o sistema deve apenas "arquivar" ou inativar o item para preservar o histórico.
  • Não expor senhas: O sistema de autenticação nunca deve exibir ou armazenar senhas de usuários em texto simples no banco de dados, nem enviá-las por e-mail.
  • Não permitir download de material restrito: O sistema não deve disponibilizar funcionalidades de download (salvar localmente) para documentos classificados como "apenas leitura online" devido a direitos autorais.
  • Não permitir buscas idênticas duplicadas: A ferramenta de cadastro não deve permitir a criação de dois registros diferentes com o mesmo ISBN/DOI, evitando duplicidade na base de dados.
  • Não aceitar formatos de arquivo proprietários: O sistema não deve aceitar o upload de documentos em formatos fechados ou proprietários que não garantam a preservação digital a longo prazo (ex: preferir PDF/A a formatos de editores de texto obscuros).
